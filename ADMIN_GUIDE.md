# 📕 관리자를 위한 운영 및 사용 매뉴얼 (Naeil)

이 문서는 '내일(Naeil)' 무장애 축제 지도 서비스를 처음 관리하게 된 운영자를 위해 작성되었습니다. 서비스의 모든 기능을 효율적으로 활용하는 방법을 상세히 안내합니다.

---

## 1. 관리자 시스템 접속 방법

보안을 위해 관리자 메뉴는 일반 사용자에게는 노출되지 않는 곳에 배치되어 있습니다.

1.  **진입 경로**: 웹사이트 어느 페이지에서나 최하단으로 스크롤을 내립니다.
2.  **링크 클릭**: 푸터(Footer) 하단 오른쪽 끝에 있는 **`[관리자 페이지]`** 글자를 클릭합니다.
3.  **인증 코드 입력**: 보안 팝업이 뜨면 아래 코드를 입력합니다.
    *   **인증 키: `naeil2025`**
4.  **최종 로그인**: 이후 나타나는 관리자 전용 로그인 화면에서도 동일한 비밀번호(`naeil2025`)를 입력하면 시스템에 완전히 접속됩니다.

---

## 2. 축제 정보 관리 (Festival Management)

관리자 페이지의 첫 번째 탭인 **[축제 관리]**에서 전체 축제 리스트를 관리합니다.

### ➕ 신규 축제 추가
*   `[+ 새 축제 추가]` 버튼을 누릅니다.
*   축제 이름, 부제목, 일시, 장소 정보를 입력합니다.
*   **썸네일 업로드**: 목록에 표시될 대표 이미지를 업로드합니다. (시스템이 자동으로 웹 최적화 압축을 진행하므로 고용량 사진도 안심하고 올리세요.)

### ✏️ 정보 수정 및 삭제
*   각 축제 카드의 `[수정]` 버튼을 눌러 내용을 바꿀 수 있습니다.
*   더 이상 운영하지 않는 축제는 휴지통 아이콘을 눌러 삭제합니다.

---

## 3. 무장애 핫스팟 설정 (Hotspot Editor)

서비스의 핵심 기능인 '지도 위 정보'를 관리하는 방법입니다. **[핫스팟 설정]** 탭을 선택하세요.

### 📍 핫스팟 추가 (지도 클릭 방식)
1.  좌측 메뉴에서 관리할 **축제를 선택**합니다.
2.  `[+ 지도 위 위치 클릭해서 추가]` 버튼을 활성화(파란색)합니다.
3.  우측 지도 이미지 위에서 실제 장소(예: 화장실, 입구 등)가 있는 위치를 **직접 클릭**합니다.
4.  클릭 즉시 편집 창이 열립니다.

### ⚙️ 핫스팟 상세 편집 (Split-View)
편집 창은 왼쪽(입력)과 오른쪽(미리보기)으로 나뉩니다.

*   **위치 및 크기 조절**: X, Y 좌표와 W(너비), H(높이) 수치를 조절하여 지도 위 핫스팟 구역의 범위를 정교하게 맞춥니다.
*   **설명 작성**: 설명란에 내용을 입력할 때 **줄바꿈(Enter)**을 하면, 사용자에게는 보기 편한 **불렛 포인트(•)** 형태로 자동 변환되어 보여집니다.
*   **이미지 섹션 (매우 중요)**:
    *   **실제 장소 이미지**: 해당 장소의 실제 모습을 보여주는 사진들을 업로드합니다.
    *   **픽토그램 이미지**: 휠체어 전용 등 해당 장소의 특성을 나타내는 아이콘 이미지를 개별적으로 업로드합니다.
*   **미리보기 활용**:
    *   **위치 미리보기**: 지도 위에서 핫스팟이 다른 장소와 겹치지 않는지 확인합니다.
    *   **팝업 상세 미리보기**: 사용자가 클릭했을 때 보게 될 **최종 디자인(사진 슬라이더 포함)**을 미리 확인하며 편집할 수 있습니다.

---

## 4. 데이터 저장 및 최적화 안내

이 서비스는 별도의 서버 DB 대신 브라우저 저장소(`localStorage`)를 적극 활용합니다.

*   **이미지 자동 압축**: 운영자가 5MB 이상의 고용량 사진을 올려도, 시스템 내부에서 품질 저하를 최소화하면서 용량을 **90% 이상 압축**하여 저장합니다. 덕분에 수백 장의 사진을 관리할 수 있습니다.
*   **저장량 체크**: 관리자 헤더 중앙의 `사용 중: 0.00 MB` 표시를 통해 현재 데이터가 얼마나 쌓였는지 실시간으로 모니터링하세요. (일반적으로 5~10MB까지 안전하게 사용 가능합니다.)
*   **데이터 주의사항**: 브라우저 캐시를 완전히 삭제하거나 시크릿 모드를 사용할 경우 등록한 데이터가 보이지 않을 수 있으니, 작업은 항상 일반 브라우저 모드에서 진행하는 것을 권장합니다.
